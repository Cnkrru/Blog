# 状态管理方案

## 项目分析

通过对项目的分析，发现当前项目使用Vue 3 + Vite + Pinia作为技术栈，已经具备了基本的状态管理架构。项目中存在以下需要状态管理的部分：

### 现有状态管理
1. **文章状态** (`stores/articles.js`)：管理文章数据的加载和获取
2. **Live2D状态** (`stores/live2d.js`)：管理Live2D小部件的显示/隐藏

### 需要状态管理的部分
1. **主题状态**：目前在App.vue和ThemeToggle.vue中分别管理，需要统一
2. **音乐播放器状态**：目前在useMusicPlayer.js中使用ref管理，需要统一
3. **工具状态**：代码运行器和串口工具的状态目前在组件内部管理
4. **全局状态**：通知、性能监控、加载状态等
5. **用户状态**：访客统计、用户位置信息等

#### 2.1 主题状态 (theme.js)

```javascript
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'

export const useThemeStore = defineStore('theme', () => {
  // 状态
  const isDark = ref(false)
  const themeMode = ref('auto') // auto, manual
  
  // 计算属性
  const currentTheme = computed(() => isDark.value ? 'dark' : 'light')
  
  // 方法
  const toggleTheme = () => {
    isDark.value = !isDark.value
    applyTheme()
    themeMode.value = 'manual'
    localStorage.setItem('themeMode', 'manual')
  }
  
  const setTheme = (dark) => {
    isDark.value = dark
    applyTheme()
  }
  
  const applyTheme = () => {
    if (typeof document !== 'undefined') {
      const body = document.body
      if (isDark.value) {
        body.classList.add('dark-theme')
      } else {
        body.classList.remove('dark-theme')
      }
      localStorage.setItem('theme', currentTheme.value)
      
      // 触发Giscus主题更新
      if (typeof updateGiscusTheme === 'function') {
        updateGiscusTheme()
      }
    }
  }
  
  const isNightTime = () => {
    const hour = new Date().getHours()
    return hour >= 13
  }
  
  const checkAutoTheme = () => {
    if (themeMode.value !== 'auto') return
    const isDarkTime = isNightTime()
    setTheme(isDarkTime)
  }
  
  const initTheme = () => {
    if (typeof document === 'undefined') return
    
    const savedTheme = localStorage.getItem('theme')
    const savedMode = localStorage.getItem('themeMode')
    
    if (savedMode) {
      themeMode.value = savedMode
    }
    
    if (themeMode.value === 'manual' && savedTheme) {
      setTheme(savedTheme === 'dark')
    } else {
      checkAutoTheme()
    }
    
    // 启动自动主题定时器
    startAutoThemeTimer()
  }
  
  let autoThemeTimer = null
  const startAutoThemeTimer = () => {
    if (autoThemeTimer) clearInterval(autoThemeTimer)
    
    autoThemeTimer = setInterval(() => {
      checkAutoTheme()
    }, 60000) // 每分钟检查一次
  }
  
  const cleanup = () => {
    if (autoThemeTimer) {
      clearInterval(autoThemeTimer)
      autoThemeTimer = null
    }
  }
  
  return {
    isDark,
    themeMode,
    currentTheme,
    toggleTheme,
    setTheme,
    initTheme,
    cleanup
  }
})
```

#### 2.2 音乐播放器状态 (music.js)

```javascript
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'

export const useMusicStore = defineStore('music', () => {
  // 状态
  const playlist = ref([])
  const currentIndex = ref(0)
  const isPlaying = ref(false)
  const isPlayerVisible = ref(false)
  const isPlaylistVisible = ref(false)
  const volume = ref(80)
  const isMuted = ref(false)
  const currentTime = ref(0)
  const duration = ref(0)
  const effectsEnabled = ref(false)
  const surroundMode = ref('off')
  const eqPreset = ref('flat')
  
  // 计算属性
  const currentSong = computed(() => {
    return playlist.value[currentIndex.value] || null
  })
  
  const progressPercent = computed(() => {
    if (duration.value === 0) return 0
    return (currentTime.value / duration.value) * 100
  })
  
  // 音频对象
  let audio = null
  let audioContext = null
  let analyser = null
  
  // 方法
  const loadMusicConfig = async () => {
    try {
      const response = await fetch('/config/music.json')
      if (!response.ok) throw new Error(`HTTP ${response.status}`)
      const config = await response.json()
      playlist.value = config.playlist || []
    } catch (error) {
      console.error('加载音乐配置失败:', error)
      playlist.value = []
    }
  }
  
  const getAudio = () => {
    if (!audio) {
      audio = new Audio()
      audio.addEventListener('timeupdate', () => {
        currentTime.value = audio.currentTime
      })
      audio.addEventListener('ended', () => {
        nextSong()
      })
      audio.addEventListener('loadedmetadata', () => {
        duration.value = audio.duration
      })
    }
    return audio
  }
  
  const getAudioContext = () => {
    if (!audioContext) {
      audioContext = new (window.AudioContext || window.webkitAudioContext)()
    }
    return audioContext
  }
  
  const getAnalyser = () => {
    if (!analyser && audioContext) {
      analyser = audioContext.createAnalyser()
      analyser.fftSize = 256
    }
    return analyser
  }
  
  const togglePlay = () => {
    const audio = getAudio()
    if (isPlaying.value) {
      audio.pause()
    } else {
      audio.play().catch(error => {
        console.error('播放失败:', error)
      })
    }
    isPlaying.value = !isPlaying.value
  }
  
  const prevSong = () => {
    currentIndex.value = (currentIndex.value - 1 + playlist.value.length) % playlist.value.length
    playCurrentSong()
  }
  
  const nextSong = () => {
    currentIndex.value = (currentIndex.value + 1) % playlist.value.length
    playCurrentSong()
  }
  
  const selectSong = (index) => {
    currentIndex.value = index
    playCurrentSong()
  }
  
  const playCurrentSong = () => {
    const song = currentSong.value
    if (song) {
      const audio = getAudio()
      audio.src = song.url
      audio.volume = volume.value / 100
      audio.play().catch(error => {
        console.error('播放失败:', error)
      })
      isPlaying.value = true
    }
  }
  
  const seek = (percent) => {
    const audio = getAudio()
    audio.currentTime = (percent / 100) * duration.value
  }
  
  const setVolume = (percent) => {
    volume.value = percent
    const audio = getAudio()
    audio.volume = percent / 100
    isMuted.value = percent === 0
  }
  
  const toggleMute = () => {
    isMuted.value = !isMuted.value
    const audio = getAudio()
    audio.volume = isMuted.value ? 0 : volume.value / 100
  }
  
  const togglePlaylist = () => {
    isPlaylistVisible.value = !isPlaylistVisible.value
  }
  
  const closePlaylist = () => {
    isPlaylistVisible.value = false
  }
  
  const applyEffects = (effects) => {
    if (effects.enabled !== undefined) {
      effectsEnabled.value = effects.enabled
    }
    if (effects.surroundMode !== undefined) {
      surroundMode.value = effects.surroundMode
    }
    if (effects.eqPreset !== undefined) {
      eqPreset.value = effects.eqPreset
    }
    // 这里可以添加音效处理逻辑
  }
  
  const cleanup = () => {
    if (audio) {
      audio.pause()
      audio = null
    }
    if (audioContext) {
      audioContext.close()
      audioContext = null
    }
    analyser = null
  }
  
  return {
    playlist,
    currentIndex,
    isPlaying,
    isPlayerVisible,
    isPlaylistVisible,
    volume,
    isMuted,
    currentTime,
    duration,
    effectsEnabled,
    surroundMode,
    eqPreset,
    currentSong,
    progressPercent,
    loadMusicConfig,
    getAudio,
    getAudioContext,
    getAnalyser,
    togglePlay,
    prevSong,
    nextSong,
    selectSong,
    seek,
    setVolume,
    toggleMute,
    togglePlaylist,
    closePlaylist,
    applyEffects,
    cleanup
  }
})
```

```javascript
export const useToolsStore = defineStore('tools', () => {

    code: 'console.log("Hello World!");\n\n// 尝试一些JavaScript代码\nfunction factorial(n) {\n  if (n <= 1) return 1;\n  return n * factorial(n - 1);\n}\n\nconsole.log("5! =", factorial(5));',
    output: '',

    executionTime: 0,
    error: null,
    stats: {
      executionCount: 0,
      totalExecutionTime: 0,
      successCount: 0,
      errorCount: 0
    }
  })
  
  // 串口工具状态
  const serialTool = ref({
    isConnected: false,
    port: '',
    baudRate: 9600,
    data: [],
    logs: [],
    isLogging: false
  })
  
  // 方法
  const updateCodeRunner = (data) => {
    codeRunner.value = { ...codeRunner.value, ...data }
  }
  
  const updateSerialTool = (data) => {
    serialTool.value = { ...serialTool.value, ...data }
  }

    serialTool.value.data.push({
      time: new Date().toISOString(),
      data
    })
    // 限制数据长度

      serialTool.value.data.shift()
    }
  }
  
  const addSerialLog = (log) => {
    serialTool.value.logs.push({
      time: new Date().toISOString(),
      log
    })
    // 限制日志长度
    if (serialTool.value.logs.length > 500) {
      serialTool.value.logs.shift()
    }
  }
  
  return {
    codeRunner,
    serialTool,
    updateCodeRunner,
    updateSerialTool,
    addSerialData,
    addSerialLog
  }
})
```

```javascript
import { defineStore } from 'pinia'
import { ref } from 'vue'

export const useGlobalStore = defineStore('global', () => {
  // 状态
  const isLoading = ref(false)
  const notifications = ref([])
  const performance = ref({
    memory: 0,
    cpu: 0,
    network: 0
  })
  
  // 方法
  const setLoading = (loading) => {
    isLoading.value = loading
  }
  
  const addNotification = (message, type = 'info', duration = 3000) => {
    const id = Date.now().toString()
    notifications.value.push({
      id,
      message,
      type,
      duration
    })
    
    // 自动移除通知
    setTimeout(() => {
      removeNotification(id)
    }, duration)
  }
  
  const removeNotification = (id) => {
    notifications.value = notifications.value.filter(notification => notification.id !== id)
  }
  
  const updatePerformance = (data) => {
    performance.value = { ...performance.value, ...data }
  }
  
  return {
    isLoading,
    notifications,
    performance,
    setLoading,
    addNotification,
    removeNotification,
    updatePerformance
  }
})
```

#### 2.5 用户状态 (user.js)

```javascript
import { defineStore } from 'pinia'
import { ref } from 'vue'

export const useUserStore = defineStore('user', () => {
  // 状态
  const visitorCount = ref(0)
  const visitorCities = ref([])
  const currentLocation = ref(null)
  
  // 方法
  const incrementVisitorCount = () => {
    visitorCount.value++
  }
  
  const addVisitorCity = (city, lat, lon) => {
    visitorCities.value.push({
      name: city,
      value: [lon, lat]
    })
  }
  
  const setCurrentLocation = (location) => {
    currentLocation.value = location
  }
  
  const fetchUserLocation = async () => {
    // API 列表，按优先级排序
    const apis = [
      {
        name: 'ip-api.com',
        url: 'https://ip-api.com/json/?fields=status,country,countryCode,city,lat,lon',
        parser: (data) => data.status === 'success' ? { city: data.city, lat: data.lat, lon: data.lon } : null
      },
      {
        name: 'ipwho.is',
        url: 'https://ipwho.is/',
        parser: (data) => data.ip ? { city: data.city, lat: data.latitude, lon: data.longitude } : null
      }
    ]
    
    let locationData = null
    
    // 依次尝试每个 API
    for (const api of apis) {
      if (locationData) break
      try {
        const response = await fetch(api.url)
        if (response.ok) {
          const data = await response.json()
          locationData = api.parser(data)
          if (locationData) {
            console.log(`${api.name} API 获取成功:`, locationData.city)
          }
        }
      } catch (error) {
        console.warn(`${api.name} API 调用失败:`, error)
      }
    }
    
    // 如果获取到位置数据，更新状态
    if (locationData && locationData.city) {
      incrementVisitorCount()
      addVisitorCity(locationData.city, locationData.lat, locationData.lon)
      setCurrentLocation(locationData)
    }
    
    return locationData
  }
  
  return {
    visitorCount,
    visitorCities,
    currentLocation,
    incrementVisitorCount,
    addVisitorCity,
    setCurrentLocation,
    fetchUserLocation
  }
})
```

### 5. 状态持久化

为了实现状态的持久化存储，我们可以使用 `pinia-plugin-persistedstate` 插件：

#### 5.1 安装插件

```bash
npm install pinia-plugin-persistedstate
```

#### 5.2 配置插件

```javascript
// stores/index.js
import { createPinia } from 'pinia'
import piniaPluginPersistedstate from 'pinia-plugin-persistedstate'

const pinia = createPinia()
pinia.use(piniaPluginPersistedstate)

export { pinia }
```

#### 5.3 在状态模块中启用持久化

```javascript
// stores/theme.js
export const useThemeStore = defineStore('theme', () => {
  // 状态定义...
  
  return {
    // 返回值...
  }
}, {
  persist: true // 启用持久化
})
```

## 优势

1. **模块化管理**：将不同类型的状态分离到不同的模块中，便于维护和管理
2. **响应式更新**：利用Vue 3的响应式系统，状态变化会自动反映到UI
3. **易于测试**：状态管理逻辑与组件逻辑分离，便于单元测试
4. **可扩展性**：可以轻松添加新的状态模块和功能
5. **持久化支持**：可以方便地实现状态的持久化存储
6. **统一管理**：集中管理所有状态，避免状态分散在各个组件中

## 总结

通过Pinia实现的状态统一管理方案，可以有效地管理项目中的各种状态，提高代码的可维护性和可扩展性。同时，结合持久化插件，可以实现状态的持久化存储，提升用户体验。

该方案适用于当前项目的需求，并且可以随着项目的发展而灵活扩展。
